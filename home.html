📖 머신러닝, 나도 러닝ㅠ (1)

드디어.. 본격적 첫 블로그 업로드하게 되었습니다!

오늘 머신러닝 논문을 읽어보니까 예전 배웠던 데이터 분석 기법들을 까맣게 잊어먹었기 때문에 지금 다시 공부해야 된다 해서 블로그 형식으로 학습노트 정리해 보겠습니다.이번에 데이터 불균형 해결과 오분류표(confusion matrix)의 기본 내용들만 정리해볼게요. 실기는 나중에 하구;;;

현실 세계의 데이터들은 항상 이상적이지 않죠. 그래서 데이터 불균형 현상 즉, 변수 분포가 어느 한쪽에 너무 치우치는 것을 의미합니다. 예전에 생각해 봤는데 우세한 class만 학습시키고 나머지는 아예 없어도 괜찮지 않을까 싶어 가지고 하지만 클래스 간 데이터 수의 차이가 너무 크면 우세한 class로 편형됩니다. 다른 말로 하면 특정의 클래스에 overfitting이 되겠죠. 아래 오분류표(confusion matrix)를 참고하고 더 쉽게 이해할 수 있을 것 같습니다.

Image source: https://velog.io/@siyeon-hwang/Confusion-Matrix

      *정확도(Accuracy): TP+TN/(TP+TN+FP+FN)
      *정밀도(Precision)/재현율(Recall): TP/(TP+FP)

우리 항상 중요하게 여겨지는 클래스의 데이터(Positive)를 많이 수집하지 못하고 있습니다. 예를 들면 불량제품보다 우량제품 더 많다는 말이죠. 이때, 더 많은 양의 데이터 보유하고 있는 클래스(일반적으로 Negative)만 학습시키면 정확도(Accuracy)가 높였지만 따라서 정밀도(Precision) 혹은 재현율(Recall)는 급격히 작아질 수 있습니다. 이런 경우 부족한 데이터를 더 수집해서 불균형 문제를 해결하면 좋겠지만 일반적으로 그렇지 못합니다. 나중에 불균형 데이터 문제를 해소하는 2가지 해결 방법을 정리해보겠습니다.
